# AI Content Writing Platform — V1.1 Development Baseline

Status: **approved for implementation**  
Target: single-user Windows local tool  
Last updated: 2026-07-18

This document overrides conflicting statements in all V1.0 planning documents.

## 1. Product and user

- The only V1 user is an independent creator operating one to three WeChat public
  accounts. It is not a team or enterprise product.
- The complete loop is: account positioning → topic → research/material → outline →
  article → review → revision → formatting/export.
- Hot-topic discovery and external search are included, but creation can start without
  a hot topic.

## 2. V1 scope

### Included

- Account and versioned account profile.
- Topics, materials, projects, outlines, versioned articles, reviews and revisions.
- URL, plain text, Markdown, DOCX and text-extractable PDF material imports.
- Hot topics from exactly these ten launch sources: Douyin, Kuaishou, Weibo, Zhihu,
  Baidu, Toutiao, The Paper, 36Kr, Huxiu and Bilibili.
- External search and webpage extraction through replaceable providers.
- Two formatting themes: `minimal` and `classic-wechat`.
- Formatting preview, copy for the WeChat editor, Markdown/HTML export and local export
  history.
- User-uploaded images stored locally. Copied content contains image placeholders; the
  user uploads images manually in the WeChat editor.
- Provider adapters for domestic and international model vendors, OpenRouter and
  OpenAI-compatible relays such as NewAPI.
- Runnable seed prompts that the local user may copy, edit, version and switch.
- Default no-login local user; optional local PIN in settings.

### Explicitly excluded

- WeChat hot topics or WeChat Index.
- AI image generation, image search and automatic image hosting.
- WeChat public-account management, direct publishing, published-article management
  and publication records.
- OCR for scanned files and Excel import.
- Team workspaces, RBAC, OIDC and SaaS deployment.
- A native Windows installer; V1 uses Docker Desktop and one-click scripts.

## 3. Version and prompt invariants

- AI writing or revision creates an immutable candidate article version.
- A candidate never becomes `current` or `primary` automatically.
- The user explicitly accepts a candidate before the current version changes.
- A prompt may have multiple active versions and at most one default active version.
- Resolution order is default active → latest active. No active version means execution
  is blocked.
- Activating a prompt version asks whether it should become the default.
- Safety-boundary instructions are maintained separately from editable user prompts.
- Historical AI generations retain their exact prompt, input, model, parameter and
  output snapshots.

## 4. Reviewers

V1 ships exactly three reviewer capabilities:

1. Account positioning and expression consistency.
2. Facts, citations and risk.
3. WeChat readability and distribution potential.

## 5. Deletion and retention

- Archive: hidden from active views and restorable.
- Ordinary delete: body becomes unavailable while minimal relationships and audit data
  remain.
- Hard delete: body, files, snapshots and encryption key are removed; a content-free
  operation record remains.
- Raw webpage snapshots default to a 14-day retention period.
- Unknown-license images are never marked publishable.

## 6. Local delivery

- Windows + Docker Desktop + Docker Compose.
- The default interface listens only on `127.0.0.1`.
- Deliver `启动平台.cmd`, `停止平台.cmd` and `备份数据.cmd`.
- The startup script checks prerequisites, starts services and opens the local app.
- Uploaded files use a local filesystem storage adapter.
- Automatic backup defaults to seven daily and four weekly recovery points.

## 7. Module order and quality gate

1. Phase 0 engineering foundation and local runtime.
2. Local identity, accounts, account profile versions and object relationships.
3. Prompt versions, model providers and generation trace.
4. Topics, materials and research.
5. Outlines, articles, reviewers, revisions and candidate acceptance.
6. Ten-source hot topics and external search.
7. Image upload placeholders, formatting, copy and export history.
8. Full security, recovery, performance, accessibility and end-to-end verification.

Every module must pass its own tests before the next module starts. V1 delivery is not
complete until the full test suite, Docker cold-start, backup/restore rehearsal and the
primary user flow all pass.
